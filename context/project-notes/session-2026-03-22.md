## Design Decisions

### DEC-008: Backup and reset as standalone commands
Safety nets for destructive experiments, upgrades, and complete dev-box removal.

### DEC-009: Environment switching uses plain directory copying
Evaluated Nix (symlinks), OCI (overlay fs), git branches (nested repo interferes with project git). Plain copying wins because context files are <50KB of text. See project-notes/architecture-env-management.md.

### DEC-010: context/shared/ for cross-environment files
User-controlled sharing via directory boundary. OWNER.md seeded in shared/ by default. Everything else in context/ is per-environment.

### DEC-011 (new): Skills + Processes architecture
Separate WHAT (process declarations in context/) from HOW (skills in SKILL.md format). Context stores artifacts (BACKLOG.md etc.), skills provide the implementation. Skill flavors enable swapping implementations (context-based vs GitHub-based). See project-notes/architecture-skills-processes.md.

## Keybinding Redesign

- **Problem:** Alt keybindings conflicted with macOS Option key (produces @, €, |, etc.)
- **Attempted:** Ctrl+g leader with Zellij Locked mode — failed (Locked mode passes through keys)
- **Solution:** Ctrl+b leader with Zellij Tmux mode — works correctly
- **Added:** Ctrl+b q as quit alternative for VS Code (which catches Ctrl+q)

## Layout Redesign

Three layouts (removed assist):
- **dev** (default): Yazi 40% + Vim 60% side by side. Tabs: claude, git, shell.
- **focus**: One tool per tab fullscreen. Tabs: files, editor, claude, git, shell.
- **cowork**: Yazi+Vim stacked left, Claude right. Tabs: git, shell.

## Research Findings

### Environment Management (DEC-009)
- uv: environment is disposable, rebuilt from lockfile. Works because .venv has no unique state.
- Nix: content-addressed store + symlinks. Assumes immutable state. Breaks in containers.
- OCI: overlay filesystem. Kernel feature, not portable to macOS.
- Git branches: nested .git confuses VS Code/lazygit/CLI git.
- Conclusion: plain copying for <50KB text files.

### MkDocs → Zensical
- MkDocs 1.x: maintenance mode since Aug 2024, support ends Nov 2026
- MkDocs 2.0: incompatible (TOML config, no plugins, no Material for MkDocs support, no license)
- Zensical: by same team, reads mkdocs.yml natively, MIT, actively developed
- Decision: migrate to Zensical

### Security Scanning
- Trivy: container image CVE scanning (free, broadest scope)
- cargo audit/deny: Rust dependency scanning
- pip-audit: Python dependency scanning
- Syft: SBOM generation (EU Cyber Resilience Act requires SBOMs)
- No Anthropic-provided supply chain tool exists

### v1.0.0 Readiness
- uv, Ruff, Zellij all remain pre-1.0 despite wide usage
- Key: stability promise (API, config schema, CLI interface)
- Checklist: versioning policy, test coverage, security audit, preview mode, real-world validation
- Realistic path: v0.5 (DX) → v0.6 (flexibility) → v0.7+ (quality) → v1.0

### Dockerfile Best Practices
- Cache mounts for apt (--mount=type=cache)
- Layer minimization
- Version pinning (digest for base images)
- Binary checksum verification
- Documented in context/work-instructions/DOCKERFILE-PRACTICES.md

### Process Frameworks
- SAFe, PMBOK, IPMA, CMMI researched (from derived project docs)
- dev-box provides levels 1-2 (meta-processes + standard templates)
- Derived projects handle levels 3-4 (tailoring + execution)
- Processes are universal, not language-specific

### SKILL.md Standard
- Open standard at agentskills.io/specification
- Progressive disclosure: metadata → instructions → resources
- Process = WHAT, Skill = HOW
- dev-box: skill installation + curated library + consistency checking
- kaits: agent orchestration + XP tracking + governance

### Tool Addons Research
- Shell tools (ripgrep, fd, bat, eza, zoxide, fzf, delta, Starship) → base image
- Infrastructure (OpenTofu, Ansible) → addon
- Kubernetes (kubectl, Helm, k9s) → addon (separate from infrastructure)
- AI providers (Claude, Aider, Gemini) → selectable, optional
- Agent orchestration frameworks → too volatile, not first-class

### Zellij Plugins
- zjstatus: configurable status bar with git/host info
- yazelix: deeper Yazi+editor integration
- Custom dev-box status plugin opportunity (WASM)

### Starship Prompt
- Clear winner over oh-my-bash, Powerlevel10k (maintenance mode)
- Rust binary, fast, cross-shell
- Has color palette system for theme coordination
- Presets: gruvbox-rainbow, catppuccin, etc.
