# Session summary

`26d3b0e`; release tag/push held for processkit integration)** and
diagnosed the underlying processkit regression that has been
masquerading as harness-side bugs for the past two releases.

The session opened with a /pk-resume morning briefing that flagged a
gap: aibox v0.18.6 had populated `.mcp.json` with all 16 processkit
MCP servers but only 4 were actually loading in the harness. Codex
made the failure mode loud (12 explicit "MCP startup failed:

paths missing the `processkit/` category segment introduced in
v0.17.0. Aibox was correctly merging them verbatim — into a broken
config — because there was no client-side validation.

   mirrored. Restores entity-layer tooling on the next session
   restart.

   [processkit#8](https://github.com/projectious-work/processkit/issues/8)
   with reproduction + suggested patch.
3. **Aibox safety rail** — `validate_script_paths()` in
   `mcp_registration.rs` would have caught this at v0.17.0 cutover
   instead of two releases later. Now hard-fails sync on any
   missing script path. Ships in v0.18.7.

Five additional fixes were rolled into the same v0.18.7 commit per
