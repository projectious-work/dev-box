# General Work Instructions

Rules and guidelines that apply to all AI agents working in this project.
This file is managed by dev-box and should not be edited directly.
Updates come via `dev-box doctor` and `dev-box update`.

## Communication

- Be concise and direct
- Lead with the answer or action, not the reasoning
- Use markdown formatting for readability

## Code Quality

- Follow existing patterns in the codebase
- Don't over-engineer or add unnecessary abstractions
- Only make changes that are directly requested or clearly necessary
- Don't add features, refactor code, or make "improvements" beyond what was asked

## Git Workflow

- Create feature branches for non-trivial changes
- Write clear, descriptive commit messages
- Don't commit secrets, credentials, or personal configuration

## Context Management

- Read CLAUDE.md before starting work
- Check context/DECISIONS.md for relevant past decisions
- Check context/BACKLOG.md for task context
- Update DECISIONS.md when making significant technical decisions
- Update BACKLOG.md when completing or starting tasks
